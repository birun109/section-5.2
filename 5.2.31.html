GIGW 3.0 Statement 5.2.31: Multiple Ways (WCAG 2.4.5)

Analysis and Requirements

**WCAG Reference****Success Criterion 2.4.5 Multiple Ways**
**WCAG Level****AA**
**Clause in Short Form**Provide Multiple Navigation Paths
**Goal**To ensure users can find all content using at least two different navigation strategies, aiding users with different cognitive needs or preferences.
**Requirement**For any non-sequential/non-process page, there must be at least two ways to find content. Common combinations are: Global Navigation + Search, or Global Navigation + Site Map.

Example for Success Criteria ‚úÖ

Success Example 1: Global Menu + Search (Most Common)

Scenario: A user needs to find the "Disability Certificate Application" form.

Method 1 (Navigation): Home | About | Services | Documents

Method 2 (Search): The user can type "Disability Certificate" into the Search box above.

**Compliance Justification:** The page offers two distinct, consistent mechanisms for navigation: **Persistent Navigation** (the Global Menu) and a **Search function**. This meets the requirement of providing "more than one way" to locate content.

Success Example 2: Global Menu + Site Map Link

Scenario: A complex website with hundreds of pages where search results can be overwhelming.

Method 1 (Navigation): Home | Departments | News | **Site Map**

Method 2 (Site Map): The user can click the "Site Map" link, which provides a hierarchical listing of all content on the website.

**Compliance Justification:** The page provides **Persistent Navigation** (Global Menu) and a **Site Map** link. A Site Map is an explicitly listed acceptable mechanism under WCAG to provide an alternative method for locating content.

Example for Failure Criteria ‚ùå

Failure Example 1: Only Sequential Navigation Provided

Scenario: A website is built with a complex, deep, hierarchical menu, but no other way to navigate.

Only Navigation: Home | Category A (12 sub-links) | Category B (20 sub-links) | Contact

The **Search box is missing**, and there is no link to a Site Map or Index.

**Compliance Justification:** While the global navigation is a valid method, it is only **one** method. A user with low vision or certain cognitive impairments may struggle to drill down through a deep hierarchy or use a screen reader to navigate dozens of menu items. **The absence of a second method (like Search or Site Map) fails SC 2.4.5.**

Failure Example 2: Exclusion Exception Misapplied

Scenario: A two-step application process has a "Review & Submit" page that features only "Previous" and "Next" buttons, but the developer incorrectly omits the global navigation and search bar.

If the **Review & Submit** page is only the final step of a **process**, omitting global navigation and search is *acceptable* (Real-time Exception). However, if the page also contains general information (like a FAQ or link to help documentation), then search/global navigation should remain to help the user locate *that* related information.

**Compliance Justification:** Developers must be careful to apply the **"result of, or a step in, a process"** exception narrowly. If a page serves any general informational purpose outside the process flow, it must still provide multiple ways to navigate the rest of the site.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 2.4.5 Multiple Ways AA Ensure that every page (except those strictly part of a transactional process) offers **at least two independent ways** for users to find other content. The most common and effective combination is **Persistent Global Navigation** (menus) **PLUS** a **Search function**.
