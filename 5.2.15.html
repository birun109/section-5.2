GIGW 3.0 Statement 5.2.15: Resize Text (WCAG 1.4.4)

Analysis and Requirements

**GIGW Statement**Text can be resized without assistive technology up to 200 percent without loss of content or functionality.
**WCAG Reference****Success Criterion 1.4.4 Resize Text**
**WCAG Level****Level AA**
**Clause in Short Form**Text Must Resize to 200% Without Loss
**Goal**To improve the ability of people with low vision to read web content by increasing text size to a readable level.
**Requirement**When text is scaled up to **200%**, content must not overlap, disappear, or become unusable.

Example for Success Criteria ✅

Success Example 1: Use of Relative Units (`em`/`rem`)

Scenario: A notice box built using relative font sizes and flexible dimensions.

This paragraph uses `rem` for font sizing and `em` for padding. If you resize the text using your browser controls (e.g., Ctrl + Plus), the box will naturally expand to accommodate the larger text, preventing clipping or overlap.

**Compliance Justification:** By using **relative units** (like `rem`, `em`, and percentages) for dimensions and font sizes, the page layout scales harmoniously with the text. The content container (`.success-box`) adapts its height and width as the text grows, ensuring **no loss of content** up to 200% text size.

Example for Failure Criteria ❌

Failure Example 1: Fixed Container Height and Width

Scenario: A small informational alert box with fixed pixel dimensions.

The container below has a **fixed height of 50px** and a **fixed width of 150px**.

This is important information that will be clipped.

**Compliance Justification:** When a user increases the text size to 200%, the text within the `.failure-box` will grow larger than the fixed 50px height. This will cause the text to **clip, overlap, or disappear**, leading to a **loss of content**. Relying on **fixed pixel units (`px`)** for containers is the primary cause of this failure.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 1.4.4 Resize Text AA **AVOID** using fixed pixel units (`px`) for font sizes, container widths, and container heights. Instead, use **relative units** (`rem`, `em`, `%`, `vh`, `vw`) to allow elements to grow and reflow when text is resized up to 200% via the browser settings.
