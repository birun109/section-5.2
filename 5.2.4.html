GIGW 3.0 Statement 5.2.4: Audio Description (WCAG 1.2.3)

Clause: Pre-recorded Synchronised Media Visual Content

Requirement: Provide an audio description track **OR** a full text alternative (transcript + visual description) for pre-recorded synchronized media.

✅ Success Example 1: Video with Audio Description (AD)

Media: A training video showing how to access a specific feature using a screen capture. The main audio only says, "Click the settings icon to proceed."

Simulated Audio Description Content (Example)

00:00:05.000 --> 00:00:08.000 (AD: The cursor moves to the top right corner, highlighting a **gear-shaped icon**.) 00:00:10.500 --> 00:00:13.000 (AD: A dropdown menu appears, listing **'Profile,' 'Help,' and 'Logout.'**)

The AD track conveys the crucial visual details (the gear icon, the menu items) that a blind user would miss if only listening to the main audio.

✅ Success Example 2: Video with Full Text Media Alternative

Media: A video announcement featuring a static presenter with key statistics appearing on screen.

Full Media Alternative (Transcript + Visual Description)

[Visual]: The screen shows a **large headline: '10% Target Achieved'** in red text.

[Speaker]: We are delighted to announce that thanks to the tireless work of our field teams, we have exceeded our initial goals.

[Visual]: The scene changes to a **map highlighting three new project locations in the Southern region**.

[Speaker]: This success allows us to immediately expand operations to three new areas, accelerating development.

The full text alternative conveys **both** the auditory and the visual information, making the content fully accessible.

❌ Failure Example: Video Missing Visual Context

Media: A policy briefing where the presenter often points to an unseen graphic or chart to make a point.

Main Audio Only: "...as you can see on the diagram, this area saw a drop-off starting last year, but we expect a full recovery by the third quarter."

The presenter references the **visual content** (the diagram) but provides no verbal description of what the diagram actually shows (the drop-off, the recovery timeline). Without an AD track or a full media alternative, the visual information is completely inaccessible to blind users.

Summary of Conformity

GIGW Statement WCAG Reference WCAG Level Required Action
5.2.4 (Audio Description) SC 1.2.3 A Provide a separate **Audio Description (AD) track** or a comprehensive **Media Alternative** that describes all visual information in the synchronized media.
