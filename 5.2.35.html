GIGW 3.0 Statement 5.2.35: Pointer Cancellation (WCAG 2.5.2)

Analysis and Requirements

**WCAG Reference****Success Criterion 2.5.2 Pointer Cancellation**
**WCAG Level****A**
**Clause in Short Form**Activation on Pointer UP
**Goal**To give users a chance to cancel an accidental click, tap, or drag action by allowing the action to finalize only upon pointer release.
**WCAG Conditions**A function must satisfy one of: **1. No Down-Event** (activation is on up-event); **2. Abort or Undo** (undo mechanism provided); **3. Up Reversal** (up-event reverses down-event outcome); **4. Essential** (down-event is critical).

Example for Success Criteria ✅

Success Example 1: Default Button Behavior (No Down-Event)

Scenario: A standard "Apply" button for submitting a form.

**Compliance Justification:** Standard HTML buttons and links are compliant by default. Their activation (click) is tied to the **up-event** (releasing the mouse button) that occurs on the same element where the down-event (pressing the mouse button) occurred. This allows a user who presses the button accidentally to move the pointer off the button before releasing it, thus **aborting the action**. (Condition 1: No Down-Event execution).

Success Example 2: Abort or Undo Mechanism

Scenario: A financial transaction button that executes immediately upon press.

*(Immediately after activation, a temporary **"Undo Transaction"** button appears next to it.)*

**Compliance Justification:** Even if the function is executed on the down-event, the page provides a prompt **"Undo"** mechanism immediately after the action is completed. This meets **Condition 2 (Abort or Undo)**, allowing the user to reverse the accidental outcome.

Example for Failure Criteria ❌

Failure Example 1: Action Triggered by Down-Event

Scenario: A developer uses JavaScript to override the default behavior and trigger a destructive action immediately on `mousedown` (down-event).

(Imagine the alert is a permanent deletion)

**Compliance Justification:** When a user with a tremor accidentally clicks this button, the action ("Delete Record") is executed the moment the mouse button is pressed (`mousedown`), giving the user **no opportunity to move the pointer away** or abort the action before completion. Since no undo mechanism is provided, this violates SC 2.5.2.

Failure Example 2: Non-Reversible Drag and Drop

Scenario: An image editor allows users to drag a layer off the canvas to delete it, but the deletion is irreversible.

Trash Can Drop Target

**Compliance Justification:** Drag-and-drop is a single pointer action. If a user accidentally drops the item into the trash area, and there is **no "Undo" function** or no clear mechanism for aborting the drop (like dropping outside an active zone), the permanent action is accidentally executed, failing SC 2.5.2.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 2.5.2 Pointer Cancellation A Ensure that all clickable/tappable controls rely on the **pointer release (up-event)** for activation. If an action *must* execute on pointer press (down-event), then you **must** provide a reliable mechanism to **Abort or Undo** the resulting action.
