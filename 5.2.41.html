GIGW 3.0 Statement 5.2.41: On Input (WCAG 3.2.2)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.2.2 On Input**
**WCAG Level****A**
**Clause in Short Form**Input Must Be Predictable
**Goal**To prevent unexpected consequences (like page navigation or submission) when a user changes a form control value, unless the user has been forewarned.
**Definition of Context Change**Major change in content, page navigation, or opening a new window.
**Requirement**A change in a form control's value (input) should not automatically cause a context change. If it does, a warning must be provided *before* the user interacts.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 3.2.2 On Input A **Rule:** A change in a form control's value (input) should **not** automatically submit a form, reload the page, or cause any other major **change of context**. If this auto-action is necessary, you **must** provide a clear, visible warning to the user *before* they change the input value.
