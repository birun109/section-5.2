GIGW 3.0 Statement 5.2.42: Consistent Navigation (WCAG 3.2.3)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.2.3 Consistent Navigation**
**WCAG Level****AA**
**Clause in Short Form**Same Order for Repeated Elements
**Goal**To provide a stable and predictable layout for navigating large websites, which is vital for users using screen readers or magnifiers.
**Requirement**Any component or group of components (e.g., links in a main menu, a set of icons in a footer) that appears on multiple pages must maintain the same relative order (source code order) wherever it appears.

Example for Success Criteria ✅

Success Example 1: Consistent Header Navigation

Scenario: The main navigation bar links are repeated across Page A and Page B.

Page A Navigation (Source Code Order)

  • (1) Home
  • (2) About Us
  • (3) Schemes
  • (4) Contact

Page B Navigation (Source Code Order)

  • (1) Home
  • (2) About Us
  • (3) Schemes
  • (4) Contact

**Compliance Justification:** The order of links (Home > About Us > Schemes > Contact) remains identical on both pages. A keyboard user or screen magnifier user who relies on link location or reading order will find the navigation predictable. This is compliant with SC 3.2.3.

Success Example 2: Adding a Link While Maintaining Relative Order

Scenario: A new link is added to the navigation, but the relative order of the original links is preserved.

Page C Navigation (Source Code Order)

  • (1) Home
  • (2) About Us
  • (NEW) Annual Report
  • (3) Schemes
  • (4) Contact

**Compliance Justification:** Although a new link ("Annual Report") was inserted, the original links still follow the same sequence: Home is still before About Us, and Schemes is still before Contact. **Insertion of new components is allowed, provided the relative order of the repeated components is unchanged.** This is compliant.

Example for Failure Criteria ❌

Failure Example 1: Swapping Link Order

Scenario: The main navigation links are swapped between Page D and Page E.

Page D Navigation (Source Code Order)

  • (1) Home
  • (2) Services
  • (3) Gallery

Page E Navigation (Source Code Order)

  • (1) Home
  • (2) Gallery
  • (3) Services

**Compliance Justification:** On Page E, the link order has changed (Gallery now comes before Services). A screen reader user, after hearing "Home," will expect "Services," but instead hears "Gallery." This unexpected shift in the repeated navigational structure causes disorientation and fails SC 3.2.3.

Failure Example 2: Swapping Global Elements

Scenario: The order of the Global Search Bar and the Language Selector is swapped between the header on Page F and Page G.

Page F Header Order: Logo -> Global Search -> Language Selector

Page G Header Order: Logo -> Language Selector -> Global Search

**Compliance Justification:** Since the Search Bar and Language Selector are repeated navigational mechanisms, their relative source code order must remain the same across the site. Swapping their position means a user tabbing through the header or relying on spatial location (e.g., using a screen magnifier) will be confused, thus failing SC 3.2.3.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 3.2.3 Consistent Navigation AA Ensure that any element or group of links that is repeated on multiple pages (e.g., primary navigation, footer links, site header components) maintains the **same relative order** in the source code on every page.
