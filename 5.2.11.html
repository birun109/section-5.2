GIGW 3.0 Statement 5.2.11: Identify Input Purpose (WCAG 1.3.5)

Analysis and Requirements

**GIGW Statement**The purpose of each input field collecting information about the user can be programmatically determined...
**WCAG Reference****Success Criterion 1.3.5 Identify Input Purpose**
**WCAG Level****Level AA**
**Clause in Short Form**Programmatically Identify Input Purpose (Autofill)
**Goal**To enable user agents (browsers, assistive technologies) to automatically fill in user data and customize the interface.
**Requirement**Input fields that collect standard user information must use the **`autocomplete` attribute** with the correct taxonomic value.

Success Example ✅

Success Example 1: Using Correct `autocomplete` Values

Scenario: A standard user registration form collecting name, email, and address.









**Compliance Justification:** Each field uses the specific, defined **`autocomplete` attribute** value (e.g., `name`, `email`, `address-line1`). This allows browsers to reliably offer **autofill** and permits assistive technologies to clearly communicate the field's purpose.

Failure Example ❌

Failure Example 1: Relying Only on `name` or `id` Attributes

Scenario: A form collecting the same information but using descriptive IDs instead of the required `autocomplete` attribute.





**Compliance Justification:** Although the labels are clear and the `id` and `name` attributes are descriptive, they are **not programmatically standardized** for autofill or purpose interpretation. A browser cannot reliably determine if `user_email` corresponds to the required WCAG value of `email`. **The `autocomplete` attribute is missing**, causing a Level AA failure.

Non-Applicable Example (Exemption)

Scenario: Input fields that do **not** collect information about the user (e.g., search terms, captchas, custom settings, new passwords).

Example: <input type="text" label="Search Query">
<input type="text" label="Captcha Code">

**Justification:** These fields are **exempt** from this requirement because their purpose is functional (searching, security) and they do not collect personal data from the standard user taxonomy. Therefore, they do not need the `autocomplete` attribute to comply with 1.3.5.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 1.3.5 Identify Input Purpose AA For any form field collecting standard user data (name, address, phone, credit card info, etc.), developers **must include the `autocomplete` attribute** with the corresponding value from the WCAG Input Purposes list to enable programmatic interpretation and autofill.
