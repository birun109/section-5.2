GIGW 3.0 Statement 5.2.46: Error Suggestion (WCAG 3.3.3)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.3.3 Error Suggestion**
**WCAG Level****AA**
**Clause in Short Form**Provide Correction Suggestion
**Goal**To give users clear, actionable instructions on how to fix a form error, preventing frustration and repeated failed submissions.
**Requirement**When an input error is detected and a correction hint is known, the error message must contain that hint, unless it compromises security (e.g., passwords or CAPTCHA).

Example for Success Criteria ✅

Success Example 1: Date Format Error with Explicit Suggestion

Scenario: User enters the date as "01-12-2025," but the system requires "YYYY-MM-DD."

Error: Invalid format. **Suggestion: Please use the suggested format: YYYY-MM-DD.**

**Compliance Justification:** The error message clearly **suggests the correct format** ("YYYY-MM-DD"). This actionable advice allows the user to correct the input without guessing, meeting the requirements of SC 3.3.3.

Success Example 2: Required Field Error with Suggestion

Scenario: User forgets to select a radio button option.

**Select Gender (Required):**
Error: Selection missing. **Suggestion: Please select one option (Male or Female).**

**Compliance Justification:** The error message not only states that a selection is missing but explicitly **suggests the correction** ("Please select one option..."). This is considered a sufficient and actionable suggestion for a required field error, making it compliant.

Example for Failure Criteria ❌

Failure Example 1: Error Identified but Correction Not Suggested

Scenario: A field requires a minimum length, but the error message only states that the input is wrong.

Error: Invalid length.

**Compliance Justification:** The error message "Invalid length" fails SC 3.3.3 because the correction is *known* (e.g., "The User ID must be at least 6 characters long"), but it is **not provided**. The user must guess the correct length or look for instructions elsewhere. This fails SC 3.3.3 (AA).

Failure Example 2: Ambiguous or Non-Actionable Suggestion

Scenario: A phone number contains letters, but the suggestion is too generic.

Error: The input is not a valid number. Please fix the mistake.

**Compliance Justification:** The message "Please fix the mistake" is not an actionable suggestion. A compliant suggestion would be, **"Suggestion: Please remove all non-digit characters from the phone number."** Since a specific suggestion for correction is known (remove letters), the failure to provide it makes this non-compliant with SC 3.3.3.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 3.3.3 Error Suggestion AA **Rule:** When a form error occurs and the means of correction is known (e.g., expected format, required field is empty), the error message **must include a clear, actionable suggestion** for how the user can fix the input (e.g., "Must be at least 8 characters," "Enter your name"). This rule applies unless the suggestion would compromise security.
