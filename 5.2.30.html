GIGW 3.0 Statement 5.2.30: Link Purpose (In Context) (WCAG 2.4.4)

Analysis and Requirements

**WCAG Reference****Success Criterion 2.4.4 Link Purpose (In Context)**
**WCAG Level****Level A**
**Clause in Short Form**Meaningful Link Text
**Goal**To ensure all users can understand where a link goes, especially when reviewing a list of all links on a page (out of visual context).
**Requirement**Link purpose must be clear from the link text itself OR from the link text combined with its programmatically determined context (e.g., the sentence, list item, or table cell it resides in).
**Developer Action**Use descriptive text in the `` tag. Avoid phrases like "read more" that are meaningless when viewed in a list.

Example for Success Criteria ✅

Success Example 1: Link Purpose Clear from Text Alone

Scenario: A direct link to a document or service.

Please download the official guidelines: **Annual Budget Report 2024 (PDF)**

**Compliance Justification:** The link text, **"Annual Budget Report 2024 (PDF)"**, is perfectly clear and descriptive on its own. It tells the user the document name, its type, and the year. This is the **best practice**.

Success Example 2: Link Purpose Clear from Context

Scenario: A series of articles or news items using the phrase "Read More."

  • Article: Latest Policy Changes for MSMEs. **Read more**.
  • Article: Deadline Extended for Income Tax Filings. **Read more**.

**Compliance Justification:** When a screen reader reads the second link, the context (the preceding list item or sentence: "Article: Deadline Extended for Income Tax Filings") is included. The full context provides the necessary purpose, thus **Read more** in this specific structure is compliant with SC 2.4.4.

Example for Failure Criteria ❌

Failure Example 1: Ambiguous Link Text

Scenario: The entire page is a list of links with meaningless text.

Available Downloads:

**Click Here** to get the form.

**Click Here** to submit the report.

**Click Here** for FAQs.

**Compliance Justification:** If a screen reader lists all links, the user hears **"Click Here, Click Here, Click Here..."** The link text alone is ambiguous, and the programmatic context (a simple preceding sentence like "Available Downloads:") does not clearly distinguish between the three links, resulting in a failure of SC 2.4.4.

Failure Example 2: Non-Descriptive "More" Links (Out of Context)

Scenario: A news widget displays three separate articles, each followed by a "More" button placed outside of the article's structured element.

Article 1: New Road Construction Project Announced

The state government has released details on the new highway...

**More**

Article 2: Drought Relief Fund Application Deadline

Farmers are advised to apply before the end of the month...

**More**

**Compliance Justification:** Because the link text is just **"More"**, and the link is not programmatically associated with the article it references (e.g., inside the same list item or table cell), a screen reader only announces **"More, More"**. The user cannot differentiate between the two links without reading the surrounding content, failing SC 2.4.4.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 2.4.4 Link Purpose (In Context) A Ensure that the **link text itself** or the **link text combined with its immediate surrounding sentence/list item/table cell** clearly communicates the destination or action of the link. **Avoid generic, context-dependent phrases** if the context is not guaranteed to be programmatically associated with the link.
