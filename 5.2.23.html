GIGW 3.0 Statement 5.2.23: Character Key Shortcuts (WCAG 2.1.4)

Analysis and Requirements

**GIGW Statement**If a keyboard shortcut is implemented in content using only letter, punctuation, number, or symbol characters, then at least one of Turn off, Remap, or Active only on focus is true.
**WCAG Reference****Success Criterion 2.1.4 Character Key Shortcuts**
**WCAG Level****Level A**
**Clause in Short Form**Single-Key Shortcuts Must Be Controllable
**Goal**To prevent accidental execution of commands by users with fine motor control issues or users of speech input software (which generates single keystrokes).
**Requirement**Global shortcuts using *only* single keys (e.g., 'S' for Save) must offer a way to **Turn Off** or **Remap** the shortcut to include a modifier (Ctrl, Alt).
**Exception**No action is required if the shortcut is **Active only when the component has focus** (e.g., typing 'M' when inside a text box).

Example for Success Criteria ✅

Success Example 1: Remap Option Provided Example Remap demo

Scenario: A complex online dashboard uses the global key 'C' to open the client history panel.

Global Shortcut: Press 'C' (single character key) to open Client History.

User Setting Provided: Located in the user's settings panel, there is an option to change the shortcut from 'C' to **'Ctrl + C'** or **'Alt + C'**.

**Compliance Justification:** The system provides a mechanism to **Remap** the shortcut to include a non-printable key (`Ctrl` or `Alt`). This satisfies the criterion because it moves the shortcut away from the easy-to-trigger single character, preventing accidental activation by speech recognition or accidental key presses.

Success Example 2: Active Only On Focus

Scenario: A video player widget uses the key 'P' to Play/Pause, but only when the video player itself is focused.

The video player only recognizes the 'P' key if the user has clicked or tabbed into the video player element itself.

Video Player Widget: Focus here (Tab into the boundary) to enable the 'P' key shortcut.

**Compliance Justification:** This is the most common and ideal solution: the shortcut is **Active only on focus**. The 'P' key does nothing when the user is simply navigating the page or typing in a normal text field. It is only active when the dedicated component has focus, meaning it does not interfere with screen readers or normal typing, and therefore meets the exception.

Example for Failure Criteria ❌

Failure Example 1: Global Single-Key Shortcut with No Override

Scenario: A public-facing government data portal implements a keyboard listener on the `document` that detects the 'M' key globally to open the main menu, and no control options are given.

Global Shortcut: Press 'M' anywhere on the page to jump to the menu.

**Compliance Justification:** If a speech input user says the word "Ma'am" or "Many," the system may process the 'M' keypress, immediately triggering the menu and disrupting their input. Since the shortcut is a single character key and is **active globally** (not just on focus), and there is **no mechanism to turn it off or remap it**, this is a direct failure of SC 2.1.4. The user cannot type the letter 'M' into the text field above without triggering the menu.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 2.1.4 Character Key Shortcuts A **AVOID** creating global keyboard shortcuts that use only a single character key (letter, number, symbol). If such a shortcut is necessary, it must satisfy one of the following: **1) Turn Off** (provide a setting to disable it), **2) Remap** (allow the user to add a modifier like Ctrl/Alt), OR **3) Restrict Scope** (ensure the shortcut is **Active only when the associated component has focus**).
