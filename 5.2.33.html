GIGW 3.0 Statement 5.2.33: Focus Visible (WCAG 2.4.7)

Analysis and Requirements

**WCAG Reference****Success Criterion 2.4.7 Focus Visible**
**WCAG Level****AA**
**Clause in Short Form**Keyboard Focus Must Be Visible
**Goal**To ensure that users navigating via keyboard always know which element has the current focus.
**Requirement**A visible focus indicator must be provided for all keyboard-operable elements. The indicator must have a minimum contrast ratio against its unfocused state.
**Developer Action**Preserve the browser's default focus ring or implement a high-contrast, custom focus indicator (like a thick outline or box shadow).

Example for Success Criteria ✅

Success Example 1: Highly Contrastive Focus Indicator

Scenario: A set of buttons uses a clear, high-contrast blue dashed outline when focused.

*(To test, press the **Tab** key to move focus onto the buttons below.)*

Link with Focus Ring

**Compliance Justification:** When focus is received, the element displays a **strong, visible change** (thick blue outline) that contrasts clearly with the surrounding unfocused state. This makes the currently focused element easy to locate, meeting SC 2.4.7.

Example for Failure Criteria ❌

Failure Example 1: Hidden Focus Indicator

Scenario: A developer uses CSS to remove the default browser focus ring globally without providing a replacement.

*(To test, press the **Tab** key to move focus onto the buttons below. You will notice the focus position is lost.)*

Link with NO Focus Ring

**Compliance Justification:** The CSS property `outline: none;` was applied to the `:focus` pseudo-class without providing any alternative visual feedback. When a keyboard user tabs to these elements, there is **no visible indication** of where the focus is located, resulting in a failure of SC 2.4.7.

Failure Example 2: Focus Indicator Hidden Behind Other Elements

Scenario: A link is placed near the edge of a container, and when focused, the focus ring is clipped by an overflow property of the container.

Link with part of the focus ring clipped or hidden. Clipped Link

**Compliance Justification:** Even if a focus indicator is technically present, if it is hidden or obscured by other content or CSS properties, it is not "visible" to the user. The user cannot fully perceive the focus location, leading to a failure of SC 2.4.7.
