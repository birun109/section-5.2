GIGW 3.0 Statement 5.2.39: Language of Parts (WCAG 3.1.2)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.1.2 Language of Parts**
**WCAG Level****AA**
**Clause in Short Form**Identify Language Changes
**Goal**To ensure assistive technologies use the correct pronunciation when encountering foreign words or phrases within content.
**Requirement**When a passage or phrase's human language differs from the page's default language (set by 3.1.1), the language of that specific part must be declared using the `lang` attribute.

Example for Success Criteria ✅

Success Example 1: Hindi Phrase within English Text

Scenario: An English document contains a quote in Hindi.

The officer greeted the delegation with a traditional phrase: **"नमस्ते"** (Namaste), before beginning the discussion. 

                <p>The officer greeted the delegation with a traditional phrase:
                    <span lang="hi">**"नमस्ते"**</span> (Namaste), before beginning the discussion.
                </p>

**Compliance Justification:** The page is English (`lang="en"` on ``). The Hindi word, **"नमस्ते"**, is correctly contained within a `` element with the `lang="hi"` attribute. A screen reader will switch to a Hindi voice for this word, ensuring correct pronunciation. This meets SC 3.1.2.

Success Example 2: French Quote (Passage) within English Text

Scenario: A large block quote in a different language.

The philosophical principle outlined was:

**"La liberté guide le peuple."** 

                <blockquote lang="fr">
                    **"La liberté guide le peuple."**
                </blockquote>

**Compliance Justification:** The entire block quote is correctly marked with `lang="fr"`. The screen reader will correctly apply French pronunciation rules to the entire sentence/passage, meeting SC 3.1.2.

Example for Failure Criteria ❌

Failure Example 1: Foreign Phrase Not Declared

Scenario: An English page contains a common foreign phrase that is not considered part of the English vernacular.

The committee agreed the decision was a **fait accompli** and moved on to the next agenda item. 

                <p>The committee agreed the decision was a **<-- lang attribute missing --> fait accompli** and moved on...</p>

**Compliance Justification:** "Fait accompli" is a French phrase. Without a `lang="fr"` declaration, an English screen reader will attempt to read it using English phonetic rules, resulting in significant mispronunciation. Although commonly used, it is generally not considered "part of the vernacular" and requires declaration. This fails SC 3.1.2.

Failure Example 2: Incorrect Language Code Used

Scenario: An English page contains a Japanese term that is incorrectly labeled as Chinese.

The term for courtesy is **rei**, which is widely practiced in Japanese culture.

                <span lang="zh">rei</span> <-- 'zh' is Chinese; word is Japanese -->

**Compliance Justification:** The word "rei" is Japanese, but the language code is set to Chinese (`zh`). The screen reader will apply Chinese pronunciation rules, leading to incorrect speech output. The determined language must accurately reflect the human language of the part, otherwise, it fails SC 3.1.2.
