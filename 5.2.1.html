GIGW 3.0 Statement 5.2.1: Non-text Content (WCAG 1.1.1)

Clause: All non-text content must have a text alternative that serves the equivalent purpose.

✅ Success Example 1: Informative Image with Equivalent ALT Text

Image: A flowchart detailing the four steps for filing an online complaint.

**Compliance Justification:** The `alt` text conveys the **full equivalent purpose and information** of the flowchart, making it accessible to screen reader users.

❌ Failure Example 1: Non-equivalent or Generic ALT Text

Image: The same flowchart detailing the four steps for filing an online complaint.

**Compliance Justification:** The `alt` text is not meaningful and **does not serve the equivalent purpose**, failing to communicate the four steps of the complaint process.

✅ Success Example 2: Control/Input (Icon Button)

Control: A magnifying glass icon used as a search button.

**Compliance Justification:** The control has a **name that describes its purpose** ("Search the Website"), as required by the Control/Input exception clause.

❌ Failure Example 2: Decorative Image Not Ignored

A decorative star separates the content.

Important announcement follows.

**Compliance Justification:** The image is pure decoration, yet the screen reader announces "Small yellow star," disrupting the flow and failing the Decoration/Formatting exception (which requires it to be implemented so it can be ignored, typically using `alt=""`).

Summary of Conformity

GIGW Statement WCAG Reference WCAG Level Required Developer Action
5.2.1 (Non-text Content) SC 1.1.1 A Provide text alternatives (`alt` attribute) that convey **equivalent purpose** for all non-text content, and use `alt=""` for purely decorative elements.
