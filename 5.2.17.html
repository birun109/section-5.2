GIGW 3.0 Statement 5.2.17: Reflow (WCAG 1.4.10)

Analysis and Requirements

**GIGW Statement**Content can be presented without loss of information or functionality and without requiring scrolling in two dimensions...
**WCAG Reference****Success Criterion 1.4.10 Reflow**
**WCAG Level****Level AA**
**Clause in Short Form**No Horizontal Scroll at 320px Width
**Goal**To ensure content uses a single vertical scrollbar at small viewports (or 400% zoom) to improve readability and usability.
**Requirement**Content must reflow into a single column when the viewport width is equivalent to **320 CSS pixels** (the width of a standard smartphone at 400% zoom).

Example for Success Criteria ✅

Success Example 1: Adaptive Column Layout (Flexbox Reflow)

Scenario: A page with two columns (main content and sidebar) that must stack vertically on small screens.

Main Content

This block uses responsive design to ensure its width is always based on the available space, **never forcing a horizontal scrollbar**. When the screen narrows, this content remains readable in a single column.

Sidebar/Navigation

This secondary block uses Flexbox to gracefully **drop beneath** the main content when the viewport width reaches 320px, eliminating horizontal scrolling and preserving all links and functionality.

**Compliance Justification:** The design successfully uses **Fluid Layouts** and **Media Queries** to adapt from a two-column structure to a single, vertically scrolling column. This allows users to read all content by scrolling only up and down, even at 400% zoom.

Success Example 2: Essential Exception (Interactive Map)

Scenario: A section displaying a large, complex, and high-detail geographic information system (GIS) map.

The map is contained within a scrollable `div`, and the main page itself does not have a horizontal scrollbar.

**Compliance Justification:** This content requires a **two-dimensional layout** for its meaning and function. Therefore, **internal horizontal scrolling** within the map component is permitted, provided the **overall page** layout does not require two-dimensional scrolling to view the surrounding text or controls.

Example for Failure Criteria ❌

Failure Example 1: Fixed Width Container

Scenario: A developer sets the width of a main content wrapper using a fixed pixel value that is greater than 320px.

The container below is styled to be **500px wide**.

This content block is 500 pixels wide. If the user views this page on a 320px width device (or zooms in 400% on a large screen), they will **be forced to scroll horizontally** to read the end of this sentence. This is the definition of two-dimensional scrolling and violates 1.4.10.

**Compliance Justification:** Using fixed pixel units (`width: 500px;`) for non-essential content forces the viewport to expand beyond 320px, introducing a **horizontal scrollbar** on the main page. This makes the content incredibly difficult to consume for users with low vision or mobile users.

Failure Example 2: Large, Non-Reflowing Images/Tables

Scenario: An embedded image or table that has a fixed width of 600px and does not scale down on small screens.

The failure is often caused by setting **`width="600px"`** directly on an `` or `

` element.

**Compliance Justification:** The large element extends beyond the 320px viewport width, creating a **horizontal scrollbar** for the entire page. All large media and data tables must either **scale down proportionally** (using relative width like `width: 100%;` and `height: auto;`) or be placed within a container that allows only one-dimensional scrolling for the element itself.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 1.4.10 Reflow AA Ensure the entire page layout is **responsive**. Content must reflow into a **single-column vertical scroll** at **320 CSS pixels** wide. **AVOID** fixed pixel widths (`px`) for large elements, containers, and main wrappers. Use **responsive design techniques** (Flexbox, Grid, Media Queries) to stack content instead of placing it side-by-side on small screens.
