GIGW 3.0 Statement 5.2.34: Pointer Gestures (WCAG 2.5.1)

Analysis and Requirements

**WCAG Reference****Success Criterion 2.5.1 Pointer Gestures**
**WCAG Level****A**
**Clause in Short Form**Avoid Complex Gestures
**Goal**To provide simple, keyboard-operable alternatives for complex touch or pointer gestures, accommodating users with motor skill limitations.
**Requirement**Any function requiring a gesture (multi-point, e.g., pinch) or a path (e.g., swipe, drag-and-drop) must also be achievable by a single click/tap (e.g., using buttons).

Example for Success Criteria ✅

Success Example 1: Map Zoom Functionality

Scenario: A government map interface where users can zoom in and out.

Interactive Map Area (Allows Pinch-to-Zoom or Scroll-Zoom)

**Compliance Justification:** The zoom function can be performed by a complex gesture (multipoint pinch on a touchscreen or path-based scroll-wheel drag on a desktop). However, the inclusion of the **"+" and "-" buttons** provides an equivalent mechanism that can be operated with a **single tap/click**, meeting SC 2.5.1.

Success Example 2: Swipe-Based Image Carousel

Scenario: A mobile news feed uses a horizontal swipe gesture to move between news slides.

Image Carousel (Swipe Left/Right)

**Compliance Justification:** The horizontal swipe (a path-based gesture) is primarily used for navigation. The inclusion of visible **"Previous"** and **"Next"** buttons allows a user to perform the exact same function with a single, non-path-based tap, making the content accessible under SC 2.5.1.

Example for Failure Criteria ❌

Failure Example 1: Path-Based Gesture Requirement (Authentication)

Scenario: A mandatory security feature requires the user to trace a complex shape (e.g., a circle) on a canvas to log in.

Login Authentication

Required: Draw the shape displayed below to proceed.

[Image of a circle to draw]

**Compliance Justification:** Tracing a specific shape is a **path-based gesture**. Since this is an authentication mechanism (not an "essential" artistic function like signing a form), and **no equivalent single-tap/click option** (like a password field or PIN entry) is provided, this fails SC 2.5.1.

Failure Example 2: Exclusive Drag-and-Drop Sorting

Scenario: A user must reorder a list of tasks using only a drag-and-drop motion (a path-based gesture) with no alternative controls.

  • Task 1 (Drag to reorder)
  • Task 2 (Drag to reorder)
  • Task 3 (Drag to reorder)

**Compliance Justification:** Dragging an item from one position to another requires a path-based gesture. If **no alternative controls** (like "Move Up" and "Move Down" buttons next to each task) are provided, users with motor impairments cannot perform the function, failing SC 2.5.1.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 2.5.1 Pointer Gestures A If your functionality requires users to perform a **path-based gesture** (swipe, drag, draw) or a **multipoint gesture** (pinch), you **must** provide an alternative mechanism that uses only a **single tap/click** (e.g., buttons, simple links) to achieve the same result.
