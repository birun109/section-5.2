GIGW 3.0 Statement 5.2.16: Images of Text (WCAG 1.4.5)

Analysis and Requirements

**GIGW Statement**If the technologies being used can achieve the visual presentation, text is used to convey information rather than images of text...
**WCAG Reference****Success Criterion 1.4.5 Images of Text**
**WCAG Level****Level AA**
**Clause in Short Form**Use Actual Text, Not Images of Text
**Goal**To ensure text is customizable, scalable, and accessible to assistive technologies.
**Requirement**Use actual text styled with **CSS** instead of rasterized images of text, unless the text presentation is **essential** (logo) or **customizable**.

Example for Success Criteria ✅

Success Example 1: Stylized Heading using CSS

Scenario: A unique, shadow-based heading style is required for branding.

Welcome to the Portal

**Compliance Justification:** The stylistic effects (unique font, large size, color, and text shadow) are achieved entirely using **CSS**. Since this is live, rendered text, a user can resize it to 200%, change its color using a browser extension, and a screen reader can read the content directly. This fulfills the criterion by prioritizing **text over image**.

Success Example 2: Essential Use (Logo)

Scenario: Displaying the official government logo containing the brand name.

**Compliance Justification:** Logos and brand names are specifically **exempt** because the particular visual presentation is **essential** for identification and branding. This is one of the valid exceptions to 1.4.5, provided the image still has proper **alt text** (fulfilling 1.1.1).

Example for Failure Criteria ❌

Failure Example 1: Navigation Links as Images

Scenario: The main navigation menu uses rasterized images for each link to achieve a specific font and color.

**Compliance Justification:** The text "Home" and "Our Services" could easily be rendered using CSS. Using image files means: 1) Users who resize text may find the text in the image does not scale, or scales poorly; 2) The text cannot be customized by a user's high-contrast theme; 3) The file size is larger than plain text. Since the presentation is **not essential**, this is a clear failure of 1.4.5.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 1.4.5 Images of Text AA **Prioritize Actual Text:** Use **HTML and CSS** to achieve all necessary styling (fonts, colors, sizes, shadows) for content, headings, and navigation. **Limit images of text** to only those instances where the visual presentation is **essential** (e.g., logos, specific diagrams) or when the user can **customize** the appearance.
