GIGW 3.0 Statement 5.2.38: Language of Page (WCAG 3.1.1)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.1.1 Language of Page**
**WCAG Level****A**
**Clause in Short Form**Set Default Page Language
**Goal**To ensure assistive technologies can correctly pronounce and translate content by knowing the dominant language of the page.
**Requirement**The primary language of the page must be declared in the opening `` tag using the `lang` attribute and a valid language code (e.g., "en", "hi", "te").

Example for Success Criteria ✅

Success Example 1: English Language Page

Scenario: A government portal page written primarily in English.

                <!DOCTYPE html>
                <html lang="en">
                    ...
                </html>

**Compliance Justification:** The use of `lang="en"` on the `` element programmatically declares that the primary language of the content is English. Screen readers will use an English voice profile for pronunciation, meeting SC 3.1.1.

Success Example 2: Hindi Language Page

Scenario: A government information page written primarily in Hindi.

                <!DOCTYPE html>
                <html lang="hi">
                    ...
                </html>

**Compliance Justification:** The use of `lang="hi"` correctly declares the primary language as Hindi. Text-to-speech software will utilize a Hindi voice, ensuring correct pronunciation and legibility for the user, meeting SC 3.1.1.

Example for Failure Criteria ❌

Failure Example 1: Language Attribute Missing Entirely

Scenario: The `lang` attribute is not included on the `` tag.

                <!DOCTYPE html>
                <html> <-- lang attribute missing -->
                    ...
                </html>

**Compliance Justification:** Without the `lang` attribute, assistive technologies cannot reliably determine the language. They may default to the user's system language, which could lead to an English screen reader trying to read Hindi content with an English accent, rendering the text incomprehensible. This fails SC 3.1.1.

Failure Example 2: Invalid or Mismatched Language Code

Scenario: A page written in English incorrectly uses the language code for French.

                <!DOCTYPE html>
                <html lang="fr"> <-- Page content is actually English -->
                    ...
                    <p>The Ministry has released the final report today.</p>
                    ...
                </html>

**Compliance Justification:** The page content is English, but the language code is set to French (`fr`). A screen reader will attempt to read the English text using a French voice profile, leading to mispronunciation and loss of meaning. While the attribute is present, the **programmatically determined language must accurately reflect the human language of the content** to pass SC 3.1.1.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 3.1.1 Language of Page A The primary language of the page **must** be declared using the `lang` attribute on the `` tag (e.g., ``). Use valid ISO 639-1 language codes.
