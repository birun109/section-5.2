GIGW 3.0 Statement 5.2.27: Bypass Blocks (WCAG 2.4.1)

Analysis and Requirements

**WCAG Reference****Success Criterion 2.4.1 Bypass Blocks**
**WCAG Level****Level A**
**Clause in Short Form**Provide Skip Navigation Link
**Goal**To allow keyboard-only users to skip directly to the unique content of a page without tabbing through repeated elements.
**Requirement**A mechanism (like a hidden link that appears on focus) must be provided as one of the very first items in the page's source code, allowing the user to bypass large, repeated content blocks.

Example for Success Criteria ✅

Success Example 1: Visually Hidden "Skip to Main Content" Link

Scenario: A website includes a standard header, search bar, and main navigation that appear on every page.

Header Block (Repeated Content): Logo | Search Bar | 10 Navigation Links

*(To test, press **Tab** once from the browser address bar. The "Skip to main content" link should appear and be focusable.)*

The first item in the HTML source code is the link: <a href="#main-content" class="skip-link">Skip to main content</a>

This is the **Main Content Area** (the target of the skip link).

**Compliance Justification:** The "Skip to main content" link is the **first focusable element** in the page source and leads directly to the primary content identified by id="main-content". The use of tabindex="-1" on the target element ensures that focus is received cleanly after the link is activated, meeting SC 2.4.1.

Example for Failure Criteria ❌

Failure Example 1: No Bypass Mechanism Provided

Scenario: A government website features a large, complex global navigation bar and a multi-level sidebar menu on every page.

**Global Navigation:** Home | Services | Documents | Help | Login (5 links)
**Sidebar Navigation:** Section 1 (8 links) | Section 2 (6 links) | Section 3 (4 links)

The main content starts directly below this massive repeated block.

This is the main content area. A keyboard user must press **Tab** 23 times to reach this area.

**Compliance Justification:** The page contains a large, repeated block of content (the navigation links). Since **no link, anchor, or programmatic method** is available to quickly jump over this block, a keyboard user or screen reader user must navigate through dozens of links on every single page they visit. This creates an enormous cognitive and physical burden, resulting in a failure of SC 2.4.1.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 2.4.1 Bypass Blocks A Implement a **"Skip to main content"** link as the **first focusable element** on every page. Ensure the link target is the start of the unique content and receives keyboard focus (e.g., by using the HTML `
` element or setting `tabindex="-1"` on the target element).
