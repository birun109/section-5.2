GIGW 3.0 Statement 5.2.40: On Focus (WCAG 3.2.1)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.2.1 On Focus**
**WCAG Level****A**
**Clause in Short Form**No Context Change On Focus
**Goal**To provide a predictable experience for all users, particularly keyboard users, by ensuring actions like navigation do not occur automatically when tabbing to an element.
**Definition of Context Change**Major change in content, page navigation, or opening a new window.
**Requirement**Focusing on an element (e.g., via the Tab key) must not automatically trigger a change of context. Such changes must be initiated by an **explicit user action** (e.g., clicks a link, presses a submit button), not merely when the control gains focus via tabbing.

Example for Success Criteria ✅

Success Example 1: Predictable Navigation

Scenario: Tabbing through a navigation menu.

Home | Services | Contact Us

**Compliance Justification:** When a user tabs from "Home" to **"Services"**, the "Services" link receives focus, but the page does not automatically navigate. The user must explicitly press **Enter/Space** or click the link to proceed. This predictability is compliant with SC 3.2.1.
