GIGW 3.0 Statement 5.2.43: Consistent Identification (WCAG 3.2.4)

Analysis and Requirements

**WCAG Reference****Success Criterion 3.2.4 Consistent Identification**
**WCAG Level****AA**
**Clause in Short Form**Same Function, Same Label
**Goal**To ensure users can confidently predict the function of a component (visually and programmatically) across an entire website, reducing errors and cognitive load.
**Requirement**Components that perform the same function must have identical **Accessible Names** (the name read by screen readers or used by voice commands) and generally identical **visual labels**.

Failure Example 1: Different Button Labels for Same Function

Scenario: Two buttons performing the exact same "next step" action use different visual labels.

Page A: Step 1 (Visual Label)

Button:

Page B: Step 2 (Visual Label)

Button:

**Compliance Justification:** Since both buttons perform the identical "move forward" function, they must share the same label. Using **"Continue"** on one page and **"Next Step"** on the next page fails to identify the component consistently. Users who look for a specific button label or use speech commands are disadvantaged. This fails SC 3.2.4.

Failure Example 2: Inconsistent Accessible Names for Icon Buttons

Scenario: A "Cart" icon button appears on multiple pages but uses different `aria-label` values.

Page C: Product Page (Accessible Name)

Icon Button:

Page D: Checkout Page (Accessible Name)

Icon Button:

**Compliance Justification:** Both buttons perform the same function (navigate to the order review page). However, the accessible name is **"Shopping Cart"** on Page C and **"View Basket"** on Page D. A speech input user who learns to say "Click Shopping Cart" will fail to activate the button on Page D. This inconsistent programmatic identification fails SC 3.2.4.

Summary of Conformity

WCAG Reference WCAG Level Key Action for Compliance
SC 3.2.4 Consistent Identification AA **Rule:** If two or more components perform the exact same action across different pages of a website, they **must** have the same **visual label** (if text is present) and the same **Accessible Name** (`aria-label` or equivalent programmatic name).
